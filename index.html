SVG

Scalable Vector Graphics

Author of presentation: Kiryukhin Viacheslav

SVG

  • What it means?
  • History

Language features

  • Description of paths
  • Description of basic geometric shapes
  • A wide range of visual properties that can be applied to shapes and paths: coloring, opacity, corner rounding, and more.
  • Interactivity
  • Animation and scripts

Advantages of the SVG format

  • Text format: SVG files can be read and edited
  • Scalability - SVG is a vector format. It is possible to enlarge any part of the SVG image without losing quality.
  • Text in SVG graphics is text, not image
  • SVG elements can be manipulated with JavaScript
  • SVG documents easily integrate with HTML documents
  • CSS compatibility. The rendering of SVG elements can be controlled with a CSS style sheet

Disadvantages of the format

  • Support for 3D graphics is completely absent
  • SVG inherits all the disadvantages of XML, such as large file size
  • Difficulty to use in large cartographic applications

SVG document structure

  • The first line is a standard XML header, a declaration indicating the XML version
    • <?xml version="1.0" encoding="UTF-8" standalone="no"?>

SVG document structure

  • The second and third lines must contain the DOCTYPE header, which determines the type of the document

    • <!DOCTYPE svg PUBLIC "-//W3C//DTD SVG 1.1//EN" "http://www.w3.org/Graphics/SVG/1.1/DTD/svg11.dtd">

    SVG document structure

    • The fourth line contains the root element of the document, specifying the SVG namespace

      • <svg version="1.1" baseProfile="full" xmlns="http://www.w3.org/2000/svg" xmlns:xlink="http://www.w3.org/1999/xlink" xmlns:ev="http://www.w3.org/2001/xml-events" width="100%" height="100%">
